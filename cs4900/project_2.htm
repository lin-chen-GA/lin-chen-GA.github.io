Project (2): Create Virtual Environment

Why create a virtual environment for each project
  1. Do not bother the default base environment
    • Some program may depend on some modules of your default base environment. By mistake, you install a different version of these modules, the program may not work any more
  2. Your project may use different version of Python, instead of the Python version in the base environment
  3. The module version your project needs is different with the version in the base environment
  4. Publish code
    • When you publish your code, a list of dependency modules will be attached. The user download your code and install your code by "python setup.py". It will download and install all the dependencies. We would like the dependency list only contains the modules your code need, instead of all modules in the base environment.
Create a virtual environment
  • The standard distribution of Python use pip + virtalenv
  • Anaconda use conda
    • conda env list # check the created virtual environment
conda create --name projectName # create a virtual environment for your project
    Start a project
    git init # add git system to the project
    Install the needed modules
    conda install scipy # install the recent version of the module scipy
conda install scipy=0.15.0 # install the specific version of the module scipy

conda list # check installed modules
conda remove scipy # remove scipy from the current virtual environment
