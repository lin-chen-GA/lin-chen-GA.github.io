IP 4: Release Project
Project Architecture

1. Create a PyPI account
2. Test Code
  • Write test code to test each function in primepackage with differnt input parameters
  • Write test code to test raises of each function in primepackage
    • 3. Add Publish Files
  • README file
    • write read file with Markdown language
    • Brief introduction of the project
    • Brief installation instruction
    • Brief user instruction
  • setup.py
  • setup.cfg
  • MANIFEST.in
  • LICENSE, choose an appropriate license
  • requirements.txt (optional)
    • 4. Release Code
  • Deposite the project to a Github repository
    • Users can download your code with "git clone URL"
    • Users can test your code with "python setup.py test"
    • Users can install the program and package with "python setup.py install"
  • Deposite the project to a PyPI respository
    • Users can install the program and package with "pip install projectName"
    • Grading Criteria
  • 1. test code, 30 points
  • 2. publish files, 30 points
  • 3. code release, 40 points
  • Make sure your code is able to be installed and run correctly, if not, you are going to receive a 0
  • The project is an individual project, if the two projects have the similarity above 90%, both projects will receive 0
    • Submission
  • Submit the URL of your source code at Github and PyPI project name to the BlazeView
